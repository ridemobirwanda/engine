┌─────────────────────────────────────────────────────────────┐
│  OPTION 2: MANUAL (STEP-BY-STEP) - 15 MINUTES              │
└─────────────────────────────────────────────────────────────┘

  STEP 1: BUILD
  > npm run build
  
  STEP 2: DEPLOY TO CLOUDFLARE
  > npm install -g wrangler
  > wrangler login
  > wrangler pages deploy dist --project-name=enginemarket
  
  STEP 3: EXPORT DATABASE
  > export-database.bat
  
  STEP 4: SETUP DATABASE
  Go to https://railway.app
  - Create MySQL database
  - Import: enginedb-cloudflare-export.sql
  - Copy connection string
  
  STEP 5: ADD ENVIRONMENT VARIABLE
  In Cloudflare dashboard:
  - Settings → Environment variables
  - Add DATABASE_URL
  - Save & Redeploy


┌─────────────────────────────────────────────────────────────┐
│  OPTION 3: VIA GITHUB - 20 MINUTES                         │
└─────────────────────────────────────────────────────────────┘

  STEP 1: BUILD & PREPARE
  > npm run build
  > deploy-dist-to-github.bat
  
  STEP 2: PUSH TO GITHUB
  Create repo at: https://github.com/new
  Name: enginemarket-dist
  
  > cd ..\enginemarket-dist
  > git remote add origin https://github.com/USERNAME/enginemarket-dist.git
  > git push -u origin main
  
  STEP 3: CONNECT CLOUDFLARE
  https://dash.cloudflare.com
  - Workers & Pages → Create → Connect Git
  - Select repository
  - Build output: / (leave empty)
  - Deploy!
  
  STEP 4: DATABASE
  Same as Option 2, Step 3-5

  🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

  ❌ dist folder not found
  → Run: npm run build
  
  ❌ Database connection failed
  → Check DATABASE_URL in Cloudflare env variables
  → Verify database allows external connections
  
  ❌ 404 on page refresh
  → Create dist/_redirects file:
     /*    /index.html    200
  
  ❌ Images not loading
  → Ensure images in public/ folder before build
  → Check image paths in code
  
  ❌ API calls failing
  → Add API_URL to environment variables
  → Deploy API separately to Railway
